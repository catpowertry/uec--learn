ue的BeginPlay事件和Tick事件
BeginPlay事件就是开始时被调用的第一次事件可以用于初始化
重点在Tick事件
Tick表示每秒运行的次数运行次数看帧率
60帧就一秒运行60次
Tick也绑定函数
例如我绑定角色在一次Tick下运行在x轴位移一个单位
那么如果你是60帧的运行状态那么你一秒就运行60个单位
那么问题来了每台电脑的运行帧率不同那怎么达成一致呢
这时候就出现了DeltaSecond参数
这个参数表示每帧间隔的时长例如第一帧和第二帧中间间隔0.1s那么DeltaSecond就是0.1Tick运行次数就是1/DeltaSecond
那么在帧率固定的情况下例如30帧和60帧
Tick1=30 Tick2=60
DeltaSecond越低帧率越高所以我只需要将高帧率的DeltaSecond*2就可以做到二者执行的Tick事件次数相同
如果我要增加角色移动速度我可以增加Tick执行次数让DeltaSecond减少就行
然后DeltaSecond的单位是秒Delta是变化量表示距离上一帧过去多少秒
例如每秒移动多少单位改成每帧移动多少单位
例如一秒移动三个单位
换成Delta就是一帧移动3*delta
上面是1*3
下面是1delta*1*3/delta    一帧×一帧所花的时间不就是时间吗

核心逻辑：不是“转化”，是“切换控制对象”，靠  PlayerController  的两个核心方法实现： UnPossess()  和  Possess() 。
1. 准备阶段：两个“身体”已经存在
 
- 你有一个战斗身体：继承自  DefaultPawn （或  ACharacter ），比如叫  MyPlayerPawn 
- 你有一个观战身体：继承自  SpectatorPawn ，比如叫  MySpectatorPawn 
 
它们都是  Pawn  的子类，继承关系在代码里写死，运行时不会变。

3. 死亡时：切换到观战身体
 
当玩家死亡，触发死亡逻辑，执行以下步骤：
 
cpp
  
void AMyPlayerController::OnPlayerDeath()
{
    // 1. 先松开当前控制的战斗身体
    UnPossess();

    // 2. （可选）隐藏/销毁战斗身体，让它从场景消失
    APawn* OldPawn = GetPawn();
    if (OldPawn)
    {
        OldPawn->SetActorHiddenInGame(true);***************这个就是把活着的角色消除
        // 或者直接销毁：OldPawn->Destroy();
    }

    // 3. 生成一个新的观战身体实例
    AMySpectatorPawn* SpectatorPawn = GetWorld()->SpawnActor<AMySpectatorPawn>(...);******************************这个跟第一开始创建的战斗身体同理

    // 4. 让控制器接管这个新的观战身体
    Possess(SpectatorPawn);
}
 
4. 关键原理： Possess()  /  UnPossess()  做了什么？
 
-  UnPossess() ：让控制器和当前 Pawn 断开连接，控制器不再控制它
-  Possess(NewPawn) ：让控制器和新的 Pawn 建立连接，开始接收输入并控制它
 整个过程没有任何“类转化”或“动态继承”，只是控制器换了一个 Pawn 实例去控制。
5. 一句话总结
通过  UnPossess()  释放旧身体，再  Possess()  新身体，实现“活着/死亡”状态切换。
